# Python 代码风格与工程约束

本文档定义 `autodev` 项目的 Python 代码风格，适用于 `autodev/` 及相关脚本。

## 1. 基本原则

1. 代码以可读性优先，命名清晰，避免过度抽象。
2. 单个函数应保持短小，逻辑层次过深时拆分辅助函数。
3. 优先显式返回与提前返回，减少嵌套。

## 2. 类型与接口

1. 新增或修改的公共函数应补充类型注解。
2. 对 `Path`、`str`、`bool` 等基础类型保持一致使用，避免隐式混用。
3. 返回值为多状态时，优先使用明确的常量或枚举语义，而不是魔法字符串。

## 3. 错误处理

1. 错误信息要可定位，至少包含关键路径或阶段信息。
2. 捕获异常后禁止静默吞掉；若降级处理，必须说明原因。
3. 用户可恢复错误用清晰日志提示，程序错误应尽早失败。

## 4. 日志规范

1. 统一使用项目既有日志方案（当前为 `loguru`）。
2. 日志消息应包含阶段信息（如计划/开发/审查/仲裁/合并）。
3. 避免输出冗余日志和重复日志。

## 5. 文件与路径

1. 使用 `pathlib.Path` 处理路径，避免字符串拼接路径。
2. 涉及计划流程时，原始 plan 文件仅可读取，不可修改。
3. 派生清单类 markdown 一律写入 `work-dir/<plan_stem>/context/`。

## 6. 文档与注释

1. 模块需有简短 docstring。
2. 复杂分支前可添加简洁注释，解释“为什么”，避免解释显而易见的“做什么”。
3. 行为变更要同步更新 README 或 docs 对应文档。
